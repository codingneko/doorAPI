actionGET

Specifies an action for the API, not used when querying, only when adding registries.

Accepted values

String (one of the following):
  • add: adds a registry to a specified door
  • addDoor: adds door

add

Adds a registry to a specified door

Required acompanying parameters:
  • status
  • doorId

addDoor

Adds a door to the database

Required acompanying parameters:
  • name
  • password

Example:
add a new registry on door 1

api.php?action=add&status=false&doorId=1

countGET

Specifies an amount of registries to return, used only when querying the database, this parameter requires no additional parameters.

Accepted values:

Integer

Example:
get the latest 10 registries

api.php?count=10

nameGET

Specifies the name for a door. This parameter is used in conjunction with the addDoor option in the action parameter and the password parameter.

Accepted values:

String (max 30 characters)


Required acompanying parameters:

  • action with addDoor value
  • password
Example:
adds a door with name "exampleDoor"

api.php?action=addDoor&password=ExamplePassword&name=exampleDoor

passwordGET

Specifies the password to use when adding a door. This parameter is used in conjunction with the addDoor option in the action parameter and the name parameter

Accepted values:

String


Required acompanying parameters:

  • action with addDoor value
  • name
Example:
adds a door using password "ExamplePassword"

api.php?action=addDoor&password=ExamplePassword&name=exampleDoor

statusGET

Specifies the status of a door. This parameter is to be used in conjunction with the add option in the action parameter and the doorId parameter.

Accepted values:

Boolean (true or false)


Required acompanying parameters:
  • action with add value
  • doorId
Example:
adds a registry to door 1 with status true, which means it's open (or it should)

api.php?action=add&doorId=1&status=true

doorIdGET

Specifies the id of a door to add a registry to. This parameter is to be used in conjunction with the add option in the action parameter and the status parameter.

Accepted values:

Integer


Required acompanying parameters:
  • action with add value
  • status
Example:
adds a registry to a door with doorId 1

api.php?action=add&status=false&doorId=1
